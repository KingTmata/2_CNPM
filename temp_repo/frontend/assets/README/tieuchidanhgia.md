## Yêu cầu báo cáo

Báo cáo cần có 5 phần chính:
1. **Giới thiệu chung**: Lý do chọn đề tài, mục tiêu, phạm vi, ý nghĩa
2. **Phân tích & thiết kế**: Personas, user stories, kiến trúc, CSDL, bảo mật, kiểm thử
3. **Quy trình cộng tác GitHub**: Sử dụng Issues, Commits, Milestones, Releases, README; nhật ký sử dụng AI có trách nhiệm
4. **Liên hệ lý thuyết Sommerville**: Mỗi quyết định kỹ thuật cần có: Lý thuyết (chương/khái niệm) → Áp dụng (minh chứng GitHub) → Phản biện
5. **Kết quả thực hiện**: Demo MVP, đánh giá, hạn chế, hướng phát triển

## Tiêu chí đánh giá chi tiết

- **Hình thức báo cáo (10%)**: Cấu trúc, định dạng, chính tả
- **GitHub (20%)**: Issues/milestones đầy đủ, commit đều, PR/code review, release, README, kiểm thử
- **Phân tích & thiết kế (20%)**: Product vision, personas, user stories, kiến trúc, kiểm thử, báo cáo chất lượng
- **Chất lượng sản phẩm (30%)**: MVP chạy được, liên hệ lý thuyết, sử dụng AI có trách nhiệm
- **Vấn đáp (20%)**: Trả lời đúng câu hỏi, giải thích quyết định kỹ thuật, đóng góp cá nhân

## Cấu trúc dự án hiện tại

Dựa trên file structure, đây là một web application bán PC/linh kiện với:
- **Backend**: ASP.NET Core API (PCZone.API)
- **Frontend**: HTML/CSS/JS với các trang: trang chủ, laptop, GPU, màn hình, build PC, giỏ hàng, thanh toán, so sánh sản phẩm, FPS calculator
- **Tính năng**: Quản lý sản phẩm, giỏ hàng, đơn hàng, thanh toán QR, AI chat, dashboard admin
- **Database**: Seed data có sẵn cho sản phẩm, danh mục, coupon
